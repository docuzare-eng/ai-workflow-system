# 门禁清单 07 - 发布与迭代

> Stage 7（发布迭代）的验收清单。分"上线前"、"上线中"、"上线后"三个阶段。

---

## 第一部分：上线前（T-24h 到 T-1h）

## A.1 部署就绪 🔴

- [ ] 所有代码已合并 main
- [ ] 所有 CI 检查绿色
- [ ] 数据库迁移已在 staging 验证
- [ ] 环境变量已配置（生产环境）
- [ ] Feature flags 已配置（如有）

## A.2 回滚就绪 🔴

- [ ] 数据库备份已做
- [ ] 回滚方案已确认
- [ ] 回滚演练已做过（项目初期）
- [ ] 有人负责监控

## A.3 监控就绪 🔴

- [ ] 关键指标 Dashboard 准备好
- [ ] 告警通知通道畅通
- [ ] 值班安排（如适用）

## A.4 沟通就绪 🟡

- [ ] 发布日志准备好
- [ ] 用户通知文案准备好（如有现有用户）
- [ ] 团队知情（如有团队）

## A.5 最终检查 🔴

- [ ] 敏感信息检查（无硬编码）
- [ ] Debug 代码已清理
- [ ] 测试数据已清理（生产库）
- [ ] 日志级别正确

---

## 第二部分：上线中（T-0 to T+1h）

## B.1 部署执行 🔴

- [ ] 数据库迁移执行成功
- [ ] 代码部署成功
- [ ] Smoke Test 通过
- [ ] 首屏可访问
- [ ] 核心功能可用

## B.2 分阶段放量（如适用） 🟡

- [ ] Canary（1%）观察 15 分钟 - 无异常
- [ ] 扩大到 10% - 无异常
- [ ] 扩大到 50% - 无异常
- [ ] 全量放开

## B.3 实时监控 🔴

- [ ] 错误率无飙升
- [ ] 响应时间无变差
- [ ] CPU/内存无异常
- [ ] 数据库无慢查询
- [ ] 关键业务指标正常

---

## 第三部分：上线后（T+1h 到 T+24h）

## C.1 持续观察 🔴

- [ ] 24 小时无 P0/P1 事故
- [ ] 错误率在可接受范围
- [ ] 用户反馈正常

## C.2 用户反馈收集 🟡

- [ ] 反馈渠道畅通
- [ ] 关键反馈已分类
- [ ] 紧急反馈已响应

## C.3 技术债登记 🟡

- [ ] 上线前的"权宜之计"已登记到 TECH_DEBT.md
- [ ] 发布日志中的 TODO 已进入待办

---

## 第四部分：持续迭代健康度

## D.1 发布节奏 🟡

- [ ] 发布频率合理（不过快不过慢）
- [ ] 每次发布有明确的价值
- [ ] 发布流程顺畅（无反复救火）

## D.2 技术债健康 🟡

- [ ] 新增 ≤ 偿还
- [ ] 高优先级债在处理
- [ ] 无持续累积

## D.3 文档同步 🟡

- [ ] R10 定期巡检
- [ ] CLAUDE.md 最新
- [ ] DECISIONS.md 持续更新
- [ ] WORKFLOW_STATE.md 实时反映现状

## D.4 风险管理 🟡

- [ ] R11 定期扫描
- [ ] 关键风险有对策
- [ ] 无被忽视的中长期风险

## D.5 事后复盘（如有事故） 🔴

- [ ] 每次事故有 Postmortem
- [ ] Postmortem 是无责备的
- [ ] 改进项有跟踪

---

## 审查决策

### 上线前
- [ ] 🟢 **可以上线**
- [ ] 🔴 **推迟上线**

### 上线中
- [ ] 🟢 **全量放开**
- [ ] 🟡 **保持小范围观察**
- [ ] 🔴 **紧急回滚**

### 上线后（24h）
- [ ] 🟢 **上线成功**
- [ ] 🟡 **有小问题需跟踪**
- [ ] 🔴 **需要 hotfix**

### 持续迭代
- [ ] 🟢 **健康**
- [ ] 🟡 **需要调整**
- [ ] 🔴 **积累过多问题需暂停开发修复**

---

> 🎯 **审查提醒**: 
> **发布不是终点，是起点。**
> **好产品是"发布了 100 次"的产品，不是"发布了 1 次完美的"产品。**
