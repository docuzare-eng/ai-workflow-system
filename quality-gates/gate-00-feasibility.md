- 本清单由 R8 项目协调官使用，审查 Stage 0 的产出

- 🟡 项强烈建议通过
- 🟢 项为加分项

---

## 一、文档完整性 🔴

- [ ] `FEASIBILITY_REPORT.md` 已创建
- [ ] `MARKET_RESEARCH.md` 已创建（如涉及市场）
- [ ] `RISK_ASSESSMENT.md` 已创建
- [ ] 所有模板占位符已填充（无 `[...]`）
- [ ] 文档格式规范（headings, tables 正确）

## 二、五维度评估完整性 🔴

- [ ] 市场维度已评估（含分数和理由）
- [ ] 技术维度已评估
- [ ] 资源维度已评估
- [ ] 战略维度已评估
- [ ] 个人维度已评估
- [ ] 每个维度的打分都有具体理由（不是空话）

## 三、决策清晰性 🔴

- [ ] 决策结论明确（🟢Go / 🟡Conditional Go / 🔴No-Go / Pivot）
- [ ] 决策理由充分（至少 3 条）
- [ ] 如是 🟡 Conditional Go，明确了"必须先解决的关键问题"

## 四、风险识别 🔴

- [ ] 已识别至少 5 个主要风险
- [ ] 每个风险有"可能性 × 影响"评估
- [ ] 每个风险有应对策略
- [ ] 关键风险（高可能 + 高影响）有详细缓解方案

## 五、数据与证据 🟡

- [ ] 市场规模估算有计算依据
- [ ] 竞品分析至少覆盖 3 个
- [ ] 关键假设有明确标记（待验证）
- [ ] 数据来源可追溯

## 六、范围与边界 🔴

- [ ] 项目目标清晰（不是模糊的"做一个 XX"）
- [ ] 目标用户明确
- [ ] 核心问题明确
- [ ] 明确排除了某些用户/场景

## 七、个人适配 🟡

- [ ] 时间投入合理（不过度乐观）
- [ ] 技能匹配度评估诚实
- [ ] 资金预算可承受
- [ ] 长期坚持意愿评估

## 八、陷阱检查 🟡

### 是否犯了以下典型错误？

- [ ] **不是** "想法很美好但无人需要"
- [ ] **不是** "技术驱动找问题"（有工具找锤子）
- [ ] **不是** "与大厂正面竞争"（除非明确差异化）
- [ ] **不是** "规模过大无法聚焦"

## 九、决策逆向检查 🟢

### 如果答案是 Go

- [ ] 想象"失败后果"能承受吗？
- [ ] 有没有更好的项目选择？
- [ ] 资源准备到位吗？

### 如果答案是 No-Go/Pivot

- [ ] 真的思考清楚了吗？（不是一时气馁）
- [ ] 转向建议是否明确？
- [ ] 能从过程中学到什么？

## 十、下一步明确性 🔴

- [ ] Stage 1 的启动方式明确（启动 R1）
- [ ] 需要准备的前置条件明确
- [ ] 谁来启动 R1 明确
